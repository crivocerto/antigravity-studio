# Memória Contínua (ClawHub Agent)

## Preferências de Arquitetura
- Frontend: Padrões de UI baseados em `frontend-design-pro` e `afrexai-nextjs-production`. Forte preferência por interfaces "Mobile-First" voltadas à conversão (estilo Link-in-Bio / Shopee / Pinterest), com Cards de destaque, preços riscados, Badges (ex: -40%), e Banners VIP fixos, sem esquecer a responsividade Web (uso de grids em Tailwind).
- Backend: Supabase com Row Level Security (RLS). Estruturas de banco de dados devem ser pensadas como multi-loja/multi-tenant (ex: Amazon, ML, Shopee) desde o princípio em sistemas de afiliação.
- CLI/Workflow: Headless puro. Autenticação via `GH_TOKEN` e `SUPABASE_ACCESS_TOKEN`. Git configurado com `ai@clawhub.com` e `ClawHub Agent`. Sem comandos interativos.
- Ferramentas: PowerShell com separação via `;` e subshell `cmd.exe /c` para `.ps1` com erro de policy.
- Hospedagem e Escalabilidade: Sistemas estáticos (Lovable) não suportam rotas de API ou Edge Functions. Para funis de Growth agressivos (Deep Linking, tracking no servidor), o projeto DEVE usar Server Components (`revalidate`) e ser hospedado na Vercel.

## Erros Passados
- **Supabase Token Parsing**: Variáveis de ambiente no PowerShell falham ao encadear `$env:VAR="x"; npx supabase` devido a erros na política de execução do npx ou perda de contexto. Solução confiável: usar `cmd.exe /c "npx supabase login --token XXX"`.
- **PowerShell vs Quotes**: Encapsular comandos longos com aspas dentro de `cmd.exe /c "..."` pode causar *Unrecognized token in source text* se não for muito bem escapado (ex: `--import-alias "@/*"`). Evite passar flags complexas por subshell se o padrão bastar.
- **Corrupção de node_modules na Migração de Pastas**: Ao mover um projeto Next.js gerado em subpasta para a raiz que já continha outro framework (ex: Vite), o uso de `xcopy` ou `move` misturado com pastas antigas corrompe as dependências (causando erro de *module not found* como o do `caniuse-lite`). **Ação corretiva definitiva**: sempre execute `rmdir /S /Q node_modules && rmdir /S /Q .next` e então um `npm install` limpo após movimentações massivas de arquivos de configuração.
- **Next.js 15+ Route Params Promise**: Ao acessar `{ params }` em `route.ts`, os parâmetros agora são assíncronos. O uso de `params.id` síncrono causa erro de build. **Ação corretiva**: Sempre use `const resolvedParams = await params;`.
- **Git Push Protection Block**: Compilar o Next.js localmente num diretório rastreado pelo Git (como `dist/` ou `.next/`) pode vazar variáveis de ambiente (`.env.local`) em arquivos de cache (`_buildManifest`, logs). Sempre garanta que a pasta de output esteja no `.gitignore` antes de rodar `git add .` se o repositório possuir segredos.
- **Playwright Stealth & Anti-Bot**: O Mercado Livre usa SPA pesado e shadow DOM no painel de afiliados, tornando seletores CSS não-confiáveis para extrair URLs geradas. **Ação corretiva**: Em bots headless (Playwright), injete cookies de sessão autêntica, ative permissões de clipboard (`['clipboard-read', 'clipboard-write']`), simule o clique no botão nativo de "Copiar" da plataforma e intercepte o link resultante via `navigator.clipboard.readText()`. Isso anula qualquer complexidade de scraping da DOM.
- **Supabase Edge Functions API Gateway (401 Unauthorized)**: Ao enviar requisições POST para Edge Functions do Supabase, o Kong API Gateway intercepta o header `Authorization`. Se você enviar um token customizado (ex: `Authorization: Bearer <minha_chave_api_bot>`), o Kong falhará ao tentar validar como JWT e retornará `401 - Invalid JWT Format`. **Ação corretiva**: Sempre envie o header `Authorization: Bearer <SUPABASE_ANON_KEY>` para passar pelo Gateway, e envie suas chaves customizadas de API em headers alternativos como `x-api-key`. No código da Edge Function, priorize a leitura desse header alternativo para validação.
- **Next.js Server Actions vs Static Export**: Server Actions que realizam validação (ex: formulários de login de Admin) causam erro no build se o `next.config.ts` possuir `output: 'export'`. **Ação corretiva**: Para áreas restritas que dependem de sessão e validação server-side, remova a diretiva `output: 'export'`.
- **Next.js Middleware Deprecation**: A partir do Next.js 16+, o arquivo `middleware.ts` foi descontinuado e passou a se chamar `proxy.ts` com a exportação `export default function proxy(request)`. **Ação corretiva**: Renomeie `middleware.ts` para `proxy.ts` para corrigir o erro `Proxy is missing expected function export name`.
- **Supabase SSR Bypass RLS**: Para deletar ou atualizar tabelas no Server Side (dentro de uma área protegida do Next.js), você deve usar a `SUPABASE_SERVICE_ROLE_KEY`. Se a chave não estiver no `.env`, extraia-a do terminal usando `cmd.exe /c "npx supabase projects api-keys --project-ref <REF>"`.
- **Supabase Unique Constraints vs Bot Clones**: Bots que rodam periodicamente podem inserir itens duplicados infinitamente. **Ação corretiva**: Em Supabase, crie uma `UNIQUE CONSTRAINT` na URL ou título (`ALTER TABLE deals ADD CONSTRAINT deals_affiliate_url_key UNIQUE (affiliate_url)`) e mude a Edge Function de `.insert()` para `.upsert(data, { onConflict: 'affiliate_url' })`.
- **Scraping de Descontos (Mercado Livre)**: Preços e descontos no Mercado Livre mudam constantemente e a matemática de markup fixo (ex: preco * 1.3) falha na realidade. **Ação corretiva**: Em bots Playwright, sempre tente extrair o preço riscado (`.andes-money-amount--previous`). Use `try/catch` para fazer fallback seguro caso não haja desconto.
- **Growth Hacks (FOMO e Expiração)**: Para gerar urgência sem sobrecarregar o cronômetro do frontend, programe limpezas automáticas no banco. **Ação corretiva**: Ative o `pg_cron` no Supabase (`CREATE EXTENSION IF NOT EXISTS pg_cron;`) para deletar `deals` antigos e garanta que o bot renove o `created_at: new Date().toISOString()` em cada `.upsert()` das Edge Functions.
- **Resiliência e Anti-Vazamento (Playwright)**: Painéis em SPA (como Mercado Livre Afiliados) são suscetíveis a timeouts. **Ação corretiva**: Sempre aplique *Exponential Backoff* (try/catch em loop de 3 tentativas com reload de página). Se o link encurtado falhar definitivamente, IGNORE a oferta e busque a próxima para não vazar tráfego orgânico não monetizado.
- **Performance de Bots (Runtime)**: Evite usar `ts-node` em produção/24h pois consome muita RAM para compilar JIT. **Ação corretiva**: Crie um script de `build: tsc` no bot e rode os arquivos via `node dist/arquivo.js` puro.
- **Auditoria Anti-Fraude (Datadome e Mercado Livre)**: APIs públicas do ML possuem proteção severa (Datadome) que bloqueia requisições "nuas e cruas" (Axios/Fetch/Node) baseando-se no TLS Fingerprint e IPs de Datacenter, retornando Erro 403 mesmo com headers mascarados. **Ação corretiva NUNCA usar API para validar preços**. Em vez disso, use o próprio Playwright para extrair o preço riscado (`original_price`) diretamente do HTML renderizado e aplique a regra de negócio localmente (ex: `((original - atual) / original) * 100` exigindo margem de 15% a 40%).
- **Arquitetura de Extração Contínua (Playwright Context Loss)**: Quando você itera sobre uma lista de ElementHandles na tela e manda o mesmo contexto navegar para outra URL (ex: Linkbuilder de afiliados), o DOM original é destruído e o laço quebra com o erro `Cannot find context`. **Ação corretiva**: Sempre use o padrão *Scrape First, Convert Later* (Duas Fases Isoladas). Salve todos os dados brutos num Array/Carrinho virtual e só use uma *nova aba* (`context.newPage()`) para as tarefas que tiram o navegador da lista original.
- **Tracking de Ineditismo (Forçando Novas Ofertas)**: Para garantir que o bot extraia apenas novos produtos diariamente sem fazer consultas caras ao banco, use um arquivo local `historico.json` persistido no disco. Adicione os títulos caçados a um `Set` global e ignore se já estiverem lá.
- **Estratégia de Caça Híbrida (Growth Hacking & Preenchimento)**: Para evitar a "Prateleira Vazia" em dias com descontos fracos, o bot deve usar um modelo híbrido: (1) **Plano A (Viral):** Usa o `google-trends-api` com "Palavras-Semente" por nicho (ex: "skincare", "base") para buscar virais no `relatedQueries`. Ele extrai ofertas virais (ex: Top 2) e envia ao DB com `is_viral: true`. (2) **Plano B (Estático):** Se a cota do nicho (ex: 4 produtos) não for preenchida, o bot faz o fallback na URL estática de ofertas gerais da categoria no Mercado Livre, ignorando itens já caçados (via `historico.json`), garantindo que a prateleira sempre fique cheia. O loop deve conter um **Filtro AdBlock** obrigatório: pular qualquer href contendo `click1.mercadolivre` ou `is_advertising=true` para barrar anúncios patrocinados que poluem o funil de descontos.
- **Padrão Visual Viral (Growth Design)**: Produtos identificados com a flag `is_viral: true` devem exibir um selo (badge) magnético na interface (ex: canto superior direito). Padrão estético obrigatório: Gradiente vibrante (ex: `from-orange-500 to-pink-500`), texto "🔥 Em Alta" e efeito de pulsação/glow (`animate-pulse shadow-[0_0_10px_rgba(...)]`) para capturar a atenção da persona vinda do TikTok.
- **Convenção Next.js 16.2.9+ (Middleware)**: A convenção clássica de `middleware.ts` foi descontinuada. Use sempre `proxy.ts` na raiz da pasta `src/` para interceptar rotas e gerenciar cookies/autenticação.
- **Resiliência de SSR em Dashboards**: Em Server Components que buscam dados (ex: via Supabase RPC `get_top_deals`), sempre envolva a requisição em blocos `try/catch` com valores padrão (fallbacks vazios). Caso o banco retorne erro ou esteja vazio, a ausência dessa proteção causa crashs silenciosos no Next.js (Router fallback) que ejetam o usuário para rotas raízes em vez de mostrar a interface de login/painel. Além disso, use `export const dynamic = 'force-dynamic';` em painéis administrativos.
